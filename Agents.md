# AGENTS.md

## Project rules
- This project is a mobile-first bail bond platform.
- Do not return partial work.
- Do not add TODO comments, demo notes, proof-of-concept text, or placeholder warnings in the UI.
- Keep the app runnable locally in seeded mode.
- Do not break local login or local testing flows.
- Keep the defendant side mobile-first and easy to use under stress.
- Keep the admin side operational, organized, and scalable.

## Brand rules
- Default brand is Angelic Bail Bonds.
- Tone must be warm, professional, supportive, discreet, and fast-response.
- Avoid tactical, militarized, aggressive, juvenile, or cheesy styling.
- Avoid generic SaaS wording.
- Use centralized brand configuration wherever practical.
- Build with future white-label / coalition branding in mind.

## UX rules
- Prioritize clarity and speed.
- Defendant dashboard should surface next court date, check-in, payment due, and contact actions first.
- Admin dashboard should surface risk, activity, reminders, and defendant status clearly.
- Large tap targets on mobile.
- Strong hierarchy, spacing, and readability.
