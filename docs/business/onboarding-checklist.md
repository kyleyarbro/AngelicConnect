# Onboarding Checklist

Use this checklist before implementation begins for a new agency.

## Agency Identity
- Legal business name
- Public-facing agency name
- Agency slug/id for platform config
- Primary service area/state coverage

## Branding Inputs
- Logo files (SVG preferred, plus PNG fallback)
- App icon source artwork
- Color preferences/brand palette
- Any typography constraints
- Approval contact for branding sign-off

## Contact and Office Details
- Primary support phone
- SMS number (if different)
- Emergency/after-hours number
- Support email
- Office address
- Office hours text

## Operational Policy Inputs
- Check-in frequency expectations
- Reminder strategy (court, payment, general)
- Escalation expectations for missed check-ins
- Payment instruction wording
- Internal note/reporting expectations

## Legal and Compliance Text
- Disclaimer language
- Privacy wording
- State-specific wording requirements (if any)
- Consent text for selfie/location check-ins

## Feature Flags and Scope
- Enabled/disabled features by agency
- Any restricted modules
- Any agency-specific workflow adjustments
- Any premium add-ons purchased

## User Provisioning
- Initial admin user list (name, role, email, phone)
- Defendant import source (if migrating)
- Role assignment confirmation
- Approval of launch access list

## Technical Inputs
- Supabase project details (if agency-specific project)
- Domain/subdomain preference
- DNS contact and access owner
- Go-live date and blackout windows

## Final Sign-Off
- Branding approved
- Config approved
- Legal text approved
- Admin training completed
- Launch approved by agency owner/decision maker
